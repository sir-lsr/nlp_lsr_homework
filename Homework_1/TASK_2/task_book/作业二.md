
Git是本地安装的软件工具（命令行或图形界面程序），用于在计算机上直接管理代码版本。
GitHub是基于Git 的代码托管平台（网站），提供远程仓库存储。

Git就像一台“智能笔记本”，专门用来记录你的代码每一步变化。比如：               
每次保存代码时，Git 会拍一张“快照”（称为 commit），记录所有文件的状态。
如果你改错代码，可以随时乘坐“时间机器”回到之前的任意快照（版本）。
还能创建平行世界（分支 branch），在新世界里尝试改动，成功后再合并到主世界。

GitHub则像一个“代码博物馆”，用来展示和共享你的时间机器快照：
你把本地的时间机器快照上传到 GitHub 博物馆（push），方便备份和分享。
其他人可以参观你的博物馆（克隆 clone），复制一份代码继续创作。
团队合作时，每个人都能在自己的时间机器里修改，最后把不同版本合并到博物馆。

Gitee：国内版GitHub，国内访问更稳定，但开源氛围不如GitHub。 


README.md是软件项目的核心说明文档（通常位于项目根目录），以轻量化的Markdown语法编写，用于简明说明项目的功能、安装步骤、使用方法等关键信息；其文件名固定为README.md，在GitHub等平台会自动渲染为美观的文档页面，是开发者了解、使用和协作项目的“初始指南”。
